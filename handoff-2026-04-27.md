### 4. Mobile hero cloud (long iteration)
Lives in the empty middle cell of the hero grid (between OLLI PAYNE and the photo). Goes through several iterations to land on:
- **SVG, not canvas** — viewBox handles scaling natively, no resize-redraw bugs from iOS Safari address-bar viewport changes
- 260 `<circle>` elements inside a `<g>`, generated by JS once at load
- CSS animation `hero-expand` on the `<svg>` root (NOT the `<g>` — iOS Safari doesn't reliably composite transforms on SVG child groups)
- 30s linear, 1 iteration, `animation-fill-mode: forwards` → expands once over 30s and freezes at peak
- `keyframes`: `0% { transform: scale(0.6); opacity: 0 } 10% { opacity: 1 } 100% { transform: scale(2); opacity: 1 }`
- Wrapped in `.hero-cloud` div: `position: absolute; inset: 0; z-index: -1` so cloud spans the whole stage behind everything
- Mobile panels now use opaque bg (`var(--bg)` inactive, `#171717` active) and `opacity: 1` (override desktop's 0.6 dim) so the cloud is hidden behind panel bodies and only peeks through the gaps between panels

**Why the SVG/CSS path mattered:** Olli kept seeing motion only during scroll. Root cause was iOS Safari throttling rAF and firing resize events during address-bar collapse. JS-driven animation appeared scroll-coupled. CSS transforms run on the compositor and keep ticking regardless.

## Quirks & gotchas

- **Preview tab is `visibilityState: hidden`.** The headless preview at localhost:8081 pauses rAF and CSS animations may freeze. Verify visual behavior in Olli's actual browser, not the preview. The preview IS good for measuring layout (`getBoundingClientRect`, computed styles, scrollWidth, etc.).
- **iOS Safari throttles rAF during scroll/idle.** Anything that needs to keep moving during scroll must be a composited CSS animation (`transform`, `opacity`) — JS rAF will look scroll-coupled even when the loop is internally fine.
- **iOS Safari fires `resize` on address-bar collapse.** Don't regenerate canvas content on resize, or the cluster will jump every time the user scrolls. Either skip the redraw, store stable positions, or use SVG (which auto-handles scaling via viewBox).
- **CSS `transform` on SVG `<g>` is unreliable in iOS Safari.** Apply transforms on the `<svg>` root or use `transform-box: fill-box` on the group.
- **Cloudflare Pages routing:** `manifestations/index.html` serves at both `/manifestations/` and `/manifestations`. Trailing-slash form is canonical for clean sub-folder URLs.
- **`text-box-trim` is vertical only.** No standard property for trimming the right side bearing of a glyph. To align "visible last pixel" of a heading with a layout edge, iterate font-size and measure `getBoundingClientRect()` on a wrapped `<span>` around the last char.
- **`box-sizing: border-box` is set globally** — `max-width` includes padding, so content area = `max-width − 2×padding`. On wide viewports `6vw` padding gets large; cap with `min(6vw, 3rem)` to prevent the content area from collapsing on 4K screens.
